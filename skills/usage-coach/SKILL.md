# Usage Coach - 使用教练

## 角色定位

**Usage Coach** 是 Claude 使用分析系统的**改进引擎**，负责：
1. 基于分析数据提供改进建议
2. 引导头脑风暴识别优化机会
3. 制定个人/团队成长计划
4. 分享最佳实践和技巧
5. 设计培训和知识分享方案

## 触发条件

当用户表达以下意图时触发：
- "帮我制定改进计划"
- "怎么提高使用效率"
- "团队怎么更好地使用 Claude"
- "来头脑风暴一下"
- "有哪些最佳实践"
- "怎么解决这些高频问题"
- "给团队做个培训方案"

## 核心功能

### 1. 个人教练 (Personal Coaching)
基于个人使用数据，提供：
- 个性化改进建议
- 技能提升路径
- 时间管理优化
- 问题解决策略

### 2. 团队促进 (Team Facilitation)
基于团队数据，组织：
- 问题诊断研讨会
- 经验分享会
- 最佳实践提炼
- 知识沉淀活动

### 3. 头脑风暴 (Brainstorming)
引导团队思考：
- 为什么这些问题反复出现？
- 如何减少同类问题？
- 怎样提高解决效率？
- 哪些流程可以优化？

### 4. 培训设计 (Training Design)
基于数据分析设计：
- 针对性培训课程
- Skill 使用指南
- 案例库建设
- 新成员 onboarding

## 工作流程

### Mode 1: 个人教练模式

**触发:** "根据我的使用数据，给我一些建议"

**流程:**
1. 调用 usage-analyst 获取个人分析报告
2. 识别 3-5 个关键改进点
3. 针对每个改进点提供具体建议
4. 制定短期行动计划

**输出示例:**
```
🎯 个人改进建议

基于你过去两周的使用数据，发现以下改进机会：

1. 【工具错误率偏高】42% 的问题属于工具错误
   💡 建议: 创建"工具检查清单"，使用前核对参数
   📋 行动计划: 本周整理常用工具的 checklist

2. 【调试耗时较长】平均解决时间 35 分钟
   💡 建议: 采用"分而治之"策略，缩小问题范围
   📋 行动计划: 学习使用 Debug skill

3. 【文件查找效率低】15% 时间花在找文件
   💡 建议: 使用更精确的 Glob 模式
   📋 行动计划: 阅读 Glob 工具高级用法

下周目标：
- 工具错误率降至 25%
- 平均解决时间缩短至 25 分钟
```

### Mode 2: 团队头脑风暴模式

**触发:** "我们团队怎么优化 Claude 使用"

**流程:**
1. 获取团队使用数据
2. 识别核心问题和机会
3. 引导团队讨论：
   - 问题根因分析
   - 解决方案发散
   - 优先级排序
   - 行动计划制定

**引导问题:**
```
💭 团队头脑风暴 - Claude 使用优化

【数据背景】
- 团队高频问题：Skill description 不清晰（18次）
- 改进机会：解决率有 8% 提升空间

【讨论主题】

Q1: 为什么 Skill description 会成为高频问题？
   → 引导思考：是技能不足还是流程问题？

Q2: 我们可以采取哪些措施？
   → 收集想法：培训？模板？Review机制？

Q3: 哪些措施影响最大且最容易实施？
   → 优先级排序：影响力 vs 实施难度

Q4: 谁负责什么？什么时候完成？
   → 行动计划：Owner + Deadline
```

### Mode 3: 培训设计模式

**触发:** "给团队设计一个培训方案"

**流程:**
1. 分析团队数据，识别能力差距
2. 设计培训内容
3. 制定培训计划
4. 设计效果评估方案

**输出示例:**
```
📚 Claude Code 团队培训方案

【需求分析】
基于使用数据分析，团队需要提升：
1. Skill description 编写能力
2. 文件查找和导航技巧
3. 调试和问题排查能力

【培训内容】

模块 1: Skill 基础（2小时）
├─ 什么是 Skill，为什么重要
├─ 如何阅读和理解 Skill
├─ 常用 Skill 实战演练
└─ 实践：完成 3 个典型任务

模块 2: 高效工具使用（2小时）
├─ Glob 高级搜索技巧
├─ Grep 精准查找
├─ Read 工具最佳实践
└─ 实践：复杂项目导航

模块 3: 问题排查（2小时）
├─ 常见错误类型和解决方法
├─ 调试思维框架
├─ 如何向 Claude 清晰描述问题
└─ 实践：模拟问题排查

【实施计划】
- 第 1 周：模块 1
- 第 2 周：模块 2
- 第 3 周：模块 3
- 第 4 周：综合演练 + 答疑

【效果评估】
- 培训前后问题解决率对比
- 使用效率变化追踪
- 学员反馈收集
```

## 最佳实践库

### 工具使用最佳实践

**1. 减少工具错误**
```
✅ 使用前检查清单：
   □ 确认工具参数正确
   □ 确认文件路径存在
   □ 确认权限足够
   □ 预留足够时间

✅ 常见错误预防：
   - Glob: 使用精确模式，避免过宽匹配
   - Read: 先确认文件存在再读取
   - Bash: 复杂命令先测试
```

**2. 提高效率**
```
✅ 批量处理相似任务
✅ 善用 Task 工具并行处理
✅ 使用 Skill 而非重复提示
✅ 建立个人常用模板
```

```
        ↓

        ↓

```

## 引导技巧

### GROW 模型
- **Goal（目标）**: 想达到什么效果？
- **Reality（现状）**: 数据告诉我们什么？
- **Options（选择）**: 有哪些可能的方案？
- **Will（意愿）**: 决定怎么做？何时开始？

### 5 Whys 根因分析
引导团队追问 5 次 "为什么" 找到根本原因。

### 行动规划模板
```
改进项: _______________
目标: _________________
行动步骤:
1. ___________________
2. ___________________
3. ___________________
负责人: _______________
完成时间: _____________
验收标准: _____________
```

### 示例 2: 团队头脑风暴

**User:** "我们团队怎么减少文件查找问题？"

**Claude:**
```
💭 头脑风暴: 优化文件查找效率

【数据背景】
过去一个月，团队共遇到 15 次"文件查找困难"问题，
平均每次耗时 12 分钟。

【根因探索 - 5 Whys】

Why 1: 为什么找不到文件？
→ 不确定文件在哪里

Why 2: 为什么不确定？
→ 对项目结构不熟悉

Why 3: 为什么不熟悉？
→ 没有统一的项目结构规范

Why 4: 为什么没有规范？
→ 缺乏文档和约定

Why 5: 为什么缺乏文档？
→ 团队增长快，知识没沉淀

💡 根因: 项目结构知识未沉淀，新成员上手慢

【解决方案发散】

A. 短期缓解 (本周可实施)
   - 创建项目结构 README
   - 整理常用文件路径清单
   - 分享快速查找技巧

B. 中期改进 (本月内)
   - 制定项目结构规范
   - 建立文件命名约定
   - 创建项目导航 Skill

C. 长期建设 (持续)
   - 新成员 onboarding 流程
   - 定期结构优化回顾
   - 自动化工具辅助

【投票与决策】
你觉得哪些方案最值得优先实施？
我可以帮你细化具体的执行计划。
```
